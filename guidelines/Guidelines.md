# FitFuel Prep 代码与设计维护准则

## 设计方向

- 保留暗黑背景、霓虹绿主行动色、橙色/蓝色营养辅助色。
- 页面优先按移动端原型展示，不引入桌面端复杂布局。
- 文案保持专业硬核健身语气，避免无关营销表达。
- 旧版浅色页面仅作为归档占位，不再扩展为主流程。

## 代码方向

- 优先基于现有 React + Vite + Tailwind 结构维护。
- 主流程集中在 `src/app/components` 的 `Dashboard`、`Profile`、`Store`、`SOP`、`Calendar`。
- 当前 demo 数据可以内联在组件中；后续如果继续扩展，再抽离到独立 data 文件。
- 不在原型阶段引入后端、真实 AI 调用或账号体系。
